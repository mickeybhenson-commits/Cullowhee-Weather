# QPF-Bias Ledger — install & operations

Quantifies Open-Meteo QPF error against MRMS MultiSensor QPE Pass2, per
sub-basin, per lead time, per season. Output feeds a guarded correction table
for the SKYE outlook tier. Fully independent of the sensor ingest path.

## File map (repo `cullowhee-weather`)

    scripts/build_mrms_masks.py    one-time, workstation (needs shapely+pyproj)
    scripts/backfill_ledger.py     one-time, workstation (needs wgrib2 for --mrms)
    ledger/ledger_db.py            shared DB module (stdlib)
    ledger/fetch_forecast.py       runtime, 6-hourly (stdlib)
    ledger/fetch_mrms.py           runtime, hourly (stdlib + wgrib2 binary)
    ledger/mrms_masks.json         GENERATED by build_mrms_masks.py — commit it
    ledger/schema_ledger.sql       canonical schema (applied idempotently)
    ledger/README_ledger.md        this file
    deploy/qpf-forecast.service    systemd
    deploy/qpf-forecast.timer      systemd (00/06/12/18:20 UTC)
    deploy/qpf-mrms.service        systemd
    deploy/qpf-mrms.timer          systemd (hourly :35)

VM data file: `/var/lib/noah/qpf_ledger.db` (WAL mode; separate from the ops
ingest DB — different lifecycle, second Litestream stanza).

## One-time setup (workstation)

    pip install shapely pyproj
    python3 scripts/build_mrms_masks.py \
        --geojson-dir /path/to/streamstats/geojsons \
        --out ledger/mrms_masks.json
    # verify: printed areas must match cullowhee_roster.csv DA to ~0.01 mi^2

Backfill (seeds ~2.5 years incl. Helene; forecasts floor is 2024-01-01
because the Open-Meteo Previous Runs precipitation archive starts there):

    sudo apt install wgrib2
    python3 scripts/backfill_ledger.py --db ./qpf_ledger.db --forecasts
    python3 scripts/backfill_ledger.py --db ./qpf_ledger.db --mrms
    # --mrms is tens of GB of transfer over many hours; interruptible,
    # restart-safe, chunkable with --start/--end
    scp qpf_ledger.db noah@<vm>:/var/lib/noah/qpf_ledger.db

## VM setup

    sudo apt install wgrib2
    sudo install -d -o noah -g noah /var/lib/noah
    sudo cp deploy/qpf-*.{service,timer} /etc/systemd/system/
    #   edit User= / WorkingDirectory= if the deployment layout differs
    sudo systemctl daemon-reload
    sudo systemctl enable --now qpf-forecast.timer qpf-mrms.timer
    # smoke test:
    sudo systemctl start qpf-forecast.service qpf-mrms.service
    sqlite3 /var/lib/noah/qpf_ledger.db \
      "SELECT source, COUNT(*) FROM forecasts GROUP BY 1;
       SELECT COUNT(*) FROM observations;"

Litestream — repo file is `deploy/litestream.yml` (covers both noah.db and
qpf_ledger.db). Deploy it to the stock service's default config path:

    sudo cp deploy/litestream.yml /etc/litestream.yml
    sudo systemctl restart litestream

Credentials (LITESTREAM_ACCESS_KEY_ID / LITESTREAM_SECRET_ACCESS_KEY) go in
the litestream service environment on the VM, never in the committed file.
If a Litestream config already exists on the VM at a different path
(check `systemctl cat litestream | grep ExecStart`), merge the qpf_ledger
stanza into that file instead and mirror the result back to the repo.

## Analysis conventions (bake into any bias fit)

* Pair via the `pairs_6h` view (6-h windows ending 00/06/12/18Z; incomplete
  windows are excluded automatically).
* WET WINDOWS ONLY: filter `qpe_mm >= 12.7` (0.5 in / 6 h) before computing
  bias — drizzle drags the multiplicative bias toward 1.0 and buries the
  orographic signal.
* Filter `min_valid_frac >= 0.8`.
* Headline metric: multiplicative bias SUM(qpf)/SUM(qpe) per
  (basin, lead bucket, season); add POD/FAR at outlook-tier thresholds.
* Keep `om-best` (live) and `om-prev-runs` (backfill) stratified — the
  backfill spans model upgrades and its issuance times are approximate
  (valid − 24 h·N).
* MRMS truth is weakest during WNC extremes (beam blockage; gauge failures
  during Helene) — cross-check major-event windows against Stage IV or
  post-event gauge analyses before letting them dominate a bias fit.

## Operational integration guard (source-ladder discipline)

No correction factor is applied to the live outlook tier until its
(basin, lead, season) cell holds at least N wet-window samples
(suggest N = 20); uncorrected QPF otherwise. Promotion is a manual step:
export the fit to a JSON the outlook tier reads (same pattern as
cn_overrides.json) — never a live query against this DB.
